# out-of-bounds updates

# Definition of [out<sub>~of</sub><sub>bounds updates](out</sub><sub>of</sub>~bounds-updates.md)
Updates done one data out<sub>~of</sub>~bounds from [Core Services](CS.md). 
_Example_
_Users updates data through ERP system own UI._

## Explanation
Out<sub>~of</sub>~bounds creates is not considered an issue if the entity only is created in one system, and therfore does not require syncronization. Out-of bounds creates will probably be syncronized if a get or save call is performed at a later stage.
