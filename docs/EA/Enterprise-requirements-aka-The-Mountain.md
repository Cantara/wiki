# Enterprise requirements (aka. The Mountain)

*Don't search for the mountain. The mountain will come to you!*

Some state and behavior is shared across several systems in the enterprise, or even outside a single enterprise. This makes affected structures harder to change. The more widely the state and behavior is shared, the harder these problems will be.

1. What issues does shared behavior and data introduce in our system?
2. How can we reduce the number of places where we have to deal with these issues?
3. What are good strategies for dealing with these strategies?
