# Antipatterns

There are some common misconceptions that greatly increase the risk of failure in a large project. These misconceptions reinforce each other and lead to "big bang" release strategies with one big release at the end of the project.

| Big bang antipatterns |
| --- |
| 1. [Feature parity](/web/20230208110804/https://wiki.cantara.no/display/ARS/Feature+parity "Feature parity") |
| 2. [Safety in rollback plans](/web/20230208110804/https://wiki.cantara.no/display/ARS/Safety+in+rollback+plans "Safety in rollback plans") |
| 3. [Painless transition](/web/20230208110804/https://wiki.cantara.no/display/ARS/Painless+transition "Painless transition") |
| 4. [Investment in outdated system is waste](/web/20230208110804/https://wiki.cantara.no/display/ARS/Investment+in+outdated+system+is+waste "Investment in outdated system is waste") |
| 5. ["It won't work here"](/web/20230208110804/https://wiki.cantara.no/pages/viewpage.action?pageId=12124244 "\"It won't work here\"") |
