# OSK - 90% av konverteringsprosjekter er meningsløs sløsing

### Intro

- Få litt mere kritiske risikomomenter opp på blokka, så man ikke driver med like stor feilrate og sløsing rundt konvertering.

### Deltakere

- Totto
- Leif Auke
- Jørgen Vasshaug  

### Postulater

- Konvertering tar ikke med seg endring i forretningsmodeller
- Teknorunking gir ikke verdi for andre enn utviklere og konsulenter (et-system strategier  er innkompatibelt med kunnskapsarbeidere - de forventer de verktøyene som gjør dem selv effektive/produktive)
- Ideen om "et system" døde før kunnskapsmedarbeideren ble født 
    - Det dreier seg ikke om systemer, men om dataflyt igjennom organisasjonen  
- Det finnes andre strategier som gjør konverteringsprosjekter overflødige
    - EDR-MDS
    - flere ...
- 50% av konverteringsprosjekter kjøres osm BigBang som er dømt til å feile uansett..
    - Forsvare 50% ...
- Man vet ikke hva fasiten er
- Feil fokus på kostnader
    - "Det koster like mye å erstatte et system som den totale kostnaden å utvikle det gamle." 
    - "Glemmer" kostnad på feiretting av nytt system, etter produksjonsetting. 
    - Hva er kostnaden på feilsøking og -retting i den teknologien man velger, kontra den teknologien man forlater?
