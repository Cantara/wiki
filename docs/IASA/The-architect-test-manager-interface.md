# The architect-test manager interface

> ℹ️ Hvilke forventninger/ønsker/krav har testleder?

Generelt sett er det luksus for en testleder å ha tilgjengelig en arkitekt som forstår sammenhengen mellom flere systemer.

Man tester som oftest hva vi kaller for "system of system"; det vil si verdikjeder men med mange system, gjerne verdikjeder som består av forskjellig typer som stormaskin, klient&Server, web og kanskje litt mobil-stæsj. Og da er der utrolig fint om man hadde en arkitekt som kan tegne og forklare hvordan dette henger sammen.

###### Ved test planlegging

- walkthrough som en review teknikk hvor målet er å at tilhørerne skal lære noe, kunne hvert greit.
- Systemskisser som gjelder for akkurat den ene greien som skal test, og ikke noe gammelt som er nesten helt riktig
- Diagrammer som state transition diagram eller flytdiagram er også veldig beskrivende og gjør det mye enklere for testeren å lag gode test case
- QA av test case, ikke alle men dem vi er usikre på

Når det gjelder testgjennomføring så er det egentlig bare å være tilgjengelig for analyse av bugs og endringsønsker samt spørsmål.
